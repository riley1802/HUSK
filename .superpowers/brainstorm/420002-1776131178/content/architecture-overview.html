Key Design Decisions

  • RAG Manager is a new layer parallel to the memory system — not inside it
  • RagToolSet exposes RAG as tools to the LLM, same pattern as MemoryToolSet
  • SqliteVectorStore behind the VectorStore interface for future ObjectBox swap
  • Embedding models managed through existing model download infrastructure
  • RAG context injection goes into system prompt alongside L1 hot memory
  • Collection ID on documents from day one (all "global" for now, ready for per-conversation later)
